# API Usage Guide: Registration & Login

No code changes needed. This explains how to use the existing APIs with your registration page.

---

## Registration Page (Name, Email, Phone)

Call `POST /Auth/Register` with all fields:

```json
{
  "Name": "Ali",
  "UserName": "ali123",
  "Email": "ali@example.com",
  "Password": "secret123",
  "ConfirmPassword": "secret123",
  "PhoneNumber": "+93786123456"
}
```

- `PhoneNumber` is optional — if provided, it's stored on the user account
- No OTP verification happens during registration (just stores the number)
- After registration, the user can log in with username+password OR phone+OTP

---

## Login Option 1: Username or Email + Password

Call `POST /Auth/login`:

```json
{
  "Username": "ali123",       // or "ali@example.com" (both work now)
  "Password": "secret123",
  "RememberMe": true
}
```

Response:
```json
{
  "status": 200,
  "data": {
    "id": 1,
    "name": "Ali",
    "userName": "ali123",
    "email": "ali@example.com",
    "phoneNumber": "+93786123456",
    "accessToken": "eyJhbGciOi...",
    "accessTokenExpires": "2026-01-20T12:00:00",
    "roles": ["User"]
  }
}
```

---

## Login Option 2: Phone + OTP Code (2 steps)

### Step 1: Send OTP code

Call `POST /Auth/SendOtpCode`:

```json
{
  "PhoneNumber": "+93786123456"
}
```

Response (success):
```json
{
  "status": "OtpSent"
}
```

User receives a 6-digit code via SMS.

### Step 2: Verify code and login

Call `POST /Auth/VerifyOtpAndLogin`:

```json
{
  "PhoneNumber": "+93786123456",
  "Code": "482917",
  "RememberMe": false
}
```

Response (success — same format as password login):
```json
{
  "status": 200,
  "data": {
    "id": 1,
    "name": "Ali",
    "userName": "ali123",
    "accessToken": "eyJhbGciOi...",
    "roles": ["User"]
  }
}
```

If the phone number is NOT registered, the user is **auto-created** with username `phone_93786123456`.

---

## Flow Summary

```
Registration:    POST /Auth/Register  (Name + Email + Password + Phone)
                        ↓
Login Method A:  POST /Auth/login     (Username/Email + Password → JWT)
                        OR
Login Method B:  POST /Auth/SendOtpCode   (Phone → SMS code)
                 POST /Auth/VerifyOtpAndLogin  (Phone + Code → JWT)
```
