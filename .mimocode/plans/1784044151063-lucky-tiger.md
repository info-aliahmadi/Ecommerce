# Convert Inventory Module to Variant Module

## Goal
Transform the basic Inventory module into a comprehensive Variant module that manages **product variants**, **variant inventory**, and **variant-attribute assignments** as standalone operations.

## Current State
- The Inventory module only does basic CRUD on `ProductInventory` (stock quantities)
- The Product module handles variants/attributes inline within `ProductService.Add/Update` but has **no standalone variant endpoints**
- Domain entities already exist: `ProductVariant`, `ProductInventory`, `ProductVariantAttribute`, `ProductInventoryTransaction`
- Models in `Hydra.Product.Core` (`ProductVariantModel`, `ProductInventoryModel`, `ProductAttributeModel`) are tightly coupled to the Product module's nested display format

## Design Decisions
1. **New models in Inventory.Core** â€” standalone variant models that flatten inventory/attributes (not nested under product)
2. **Keep Product module unchanged** â€” ProductService.Add/Update continue creating variants inline; no breaking changes
3. **Reuse existing domain entities** â€” no EF migration needed
4. **New permission constant** â€” `SALE_PRODUCT_VARIANT_MANAGEMENT` (add to `EcommercePermissionTypes`)

### 3. Service â€” `Hydra.Inventory.Api/Services/`

**Delete:** `InventoryService.cs`

**Create:** `VariantService.cs`
- Implements `IVariantService`
- Queries `ProductVariant` entity with includes for `ProductInventory`, `VariantAttributes.Attribute`
- Flatten inventory and attributes into `VariantModel`
- Add/Update: create/update `ProductVariant` + `ProductInventory` + `ProductVariantAttribute` entries in a transaction
- UpdateStock: update `ProductInventory` stock and create a `ProductInventoryTransaction` record
- Delete: remove variant, its inventory, variant attributes, and inventory transactions
